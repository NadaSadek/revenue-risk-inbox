# Project instructions

This is a Next.js + TypeScript portfolio project focused on AI-assisted revenue-risk analysis for SaaS support workflows.

## Priorities

- Prefer simple, explicit code over clever abstractions.
- Do not add new libraries unless there is a clear reason.
- Preserve existing UI copy unless asked to change it.
- Do not invent backend behavior that is not implemented.
- Keep AI/schema logic in `lib/ai`.
- Keep route handlers in `app/api`.
- Validate data at boundaries with Zod.
- Run typecheck before claiming work is complete.

## Style

- Use TypeScript strictly.
- Avoid `any`.
- Prefer small pure helper functions for business logic.
- Components should stay readable and not over-abstracted.
- Use Tailwind utility classes consistently.

## Validation

Before finishing:

- explain changed files
- mention tradeoffs
- mention anything left incomplete
